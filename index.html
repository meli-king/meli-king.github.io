Completed Projects

What I've completed. These include both projects done for school and personal passion projects.

RISC-V Unit Tester

May 2022 - August 2022 | Utilizes Java, JUnit, Assembly (RISC-V and MIPS)

Together with a group of three other students working under Professor Andrew Petersen, we created a program to test student submitted assembly code automatically. Based off of MIPSUnit, a unit testing framework for MIPS assembly. The goal of the tester was to provide an easier way for the teaching staff of the Computer Architecture course to test student code.

In this, I learned how to simulators work, how to work with a large, preexisting repository of code, and how to effectively communicate with multiple people as a team.

Linux Shell Program

January 2022 - April 2022 | Utilizes C, Python, pytest

A C program containing most of the basic functionality of a shell like Bash. This shell can handle simple commands like echo and variable declaration, while also being able to chain commands via pipes and send signals and text to other shell instances. Python and pytest was used to test the correctness of the shell. The goal of this shell was to utilize and apply knowledge about system calls in a practical application.

In this, I learned about the behaviour of a shell, how use system calls and how to build a large project from the ground up.

Simon

March 2022 | Utilizes Assembly (RISC-V)

The original game of Simon, but made entirely in assembly. The player must input a button sequence identical to the order and colour the buttons were originally flashed. As the player progresses, the speed and the amount of colours in the sequence increases. The game ends when the player fails to provide a correct sequence of colours. The goal of this game was to learn how to write assembly code and utilize the heap using system calls. UI provided through Ripes, a visual computer architecture simulator and assembly code editor.

In this, I learned how to code in assembly, use system calls and utilize functions that control external devices.

Three Musketeers

September 2021 - December 2021 | Utilizes Java, JavaFX, Scrum/Agile Methodologies, Git

Together with a group of three other students, we recreated the classic game of The Three Musketeers using Java, and a GUI made with JavaFX. Development of this game involved the use of UML, Git flow and Scrum Methodologies. The main goal of the group project was to learn how to collaborate in a group similar to a real world setting.

In this, I learned about UML diagrams, Scrum Methodologies, and how to collaborate with new people effectively.

Image Compression Algorithm

March 2021 | Utilizes Python, Object Oriented Programming

A program that can take a given image and produce a corresponding jpeg file of the same image but at a lower quality. Conversely, the program can also take a compressed image and expand it back to the original image. The goal of this compressor was to apply concepts of object oriented design and practice recusrion.

In this, I learned how to utilize object oriented design, recursion and file IO.

GUI Interactive Tutorial for Computer Hardware

January 2020 | Utilizes Java, Processing3, JavaScript, P5

In a group with two other students, we created a program that would introduce and teach students about computer hardware components. The program GUI was provided by Processing3 and consisted of three parts:

  1. A brief lesson, showing an image of each component and the purpose of it
  2. A drag and drop portion, where the students can "build" a PC by dragging and dropping the components into an empty case
  3. A quiz, where students can test their knowledge and see how much they learned

Additionally, a teacher asked me to remake the project in P5 (Processing5) which uses JavaScript instead of Java. The goal was to imbed the P5 project into the course website.

In this, I learned the basics of object oriented design, how to work in a small group and some JavaScript.

Wave Racer

June 2019 | Utilizes Java, Processing3

A simple game where the player controls a speedboat and must avoid incoming obstacles in the water as they race the waves. As the player progresses, the speed of the speedboat increases and the frequency of obstacles increases. Once the boat collides with an obstacle, the game ends and the player is given a score. The goal of the project is to apply all programming concepts learned so far and program a simple game using Java.

In this, I learned about arrays, programming in more modular ways, and working with multiple sprites.
